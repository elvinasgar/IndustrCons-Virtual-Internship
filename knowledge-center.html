Why cube strength tests don't catch every concrete defect

A compressive strength test measures exactly one thing: how much load a cured sample can carry before failure. It says nothing about durability under chemical exposure — sulfate attack, chloride ingress, carbonation — because none of those mechanisms show up within a 28-day strength test window. A mix can pass strength testing at 35 MPa and still fail a durability requirement if the wrong cement type was used. This is why specifications separate strength grade (e.g. C35) from exposure/durability class (e.g. sulfate-resisting) — they're independent requirements verified by different tests, on different timelines.

Reading a footing schedule without guessing

A footing schedule packs a lot into a small table: plan dimensions, depth, reinforcement diameter and spacing (often written as "16mm @ 150 c/c"), cover requirements, and concrete grade. The order to read it in is dimensions → reinforcement → cover → grade, because each later item depends on context from the one before it — cover, for instance, is measured differently to a formed face than to blinding, so you need to already know which face you're checking before the cover number means anything.

Why "round up" isn't always the safe answer

Rounding direction depends on what you're solving for. Checking maximum spacing compliance (like rebar centers) → round up, because rounding down could leave a gap that exceeds the limit. Estimating a cost quantity from a take-off → the correct convention is usually set by the BOQ measurement rules, which may round to whole units, nearest 0.5, or not round mid-calculation at all. Treat "round up" as a rule for one specific kind of question, not a universal habit.

What actually belongs on a pre-pour checklist

A pour-readiness checklist should map one-to-one to the method statement's defined sequence for that specific pour — nothing more, nothing less. Items that are legitimate site tasks but aren't part of the pour sequence (delivery reconciliation, forecast reviews, unrelated administrative sign-offs) dilute the checklist and slow down genuine readiness sign-off. If an item isn't gating the pour, it belongs in a different log.
